Five-minute task recipes

Send a one-off email

  1. g 1 · c · type name · ↵
  2. Set Subject + From in the right rail
  3. Build the body in the Email Builder tab — drag modules, set tokens like {{contact.firstName}}
  4. Add recipients · click Send test · review
  5. Send now or set Scheduled Date

Build a 3-step nurture sequence

  1. g 3 · c · name it · ↵
  2. Cadence tab → add Email step → wait 3 days → email step → wait 5 days → email step
  3. Set Entry trigger (form submission / list)
  4. Status → Active after testing on a small list

Capture leads with a form

  1. g 4 · c · name it · ↵
  2. Add fields — Type dropdown has 16 options: text, long text, number, boolean, rating, date, date+time, email, phone, URL, address, full name, file, select, multi-select, record
  3. Toggle Auto-create person; optionally set Tags to apply on submission (segments contacts by which form they used)
  4. Publish · copy the embed iframe — or POST your own form to /forms/<workspace-id>/<form-id>/submit (CORS open, no auth)

Stop bots without signing up for anything

Six bot-protection layers in form Settings tab. Five are native (zero accounts, zero API keys); only Turnstile is a third-party service. Recommended baseline:

  1. Honeypot field name: company_url — then add an off-screen <input name="company_url" tabindex="-1" style="position:absolute;left:-9999px"> to your form. Bots fill it; humans don't.
  2. Require form-load token on with Minimum submit time 2 — defeats direct-POST scrapers + instant-fire replays
  3. Rate limit 5 — caps per-IP submissions at 5/min
  4. Allowed origins https://your-domain.com — rejects cross-site spam
  5. Optional: Reject disposable emails, Cloudflare Turnstile for high-volume forms

Honeypot is not a service. No sign-up, no API key. Just a hidden field.

Spin up a landing page

  1. g 7 · c · title it · ↵
  2. Settings tab — set the slug (e.g. spring-launch)
  3. Builder tab — click a layout (1 col, 2 col…) on the left rail to add a section
  4. Click module tiles (Hero, Heading, Button, Form embed…) to add them
  5. Click any module on the canvas to edit it in the right rail (colors, alignment, content)
  6. Status → PUBLISHED — page goes live at /lp/<slug>

Find what you were just working on

  1. ⌘ K
  2. The "Today" group at the top shows records you visited in the last 24 hours
  3. Click the one you want — or arrow + ↵

Pin a record for one-click access later

  1. Open the record
  2. Press p — toast confirms "Pinned"
  3. Next time, jump back with 1–9 (depending on its position)

Landing-page module library

Layout modules

  • Header / Nav — brand + links + CTA
  • Hero — full-width banner, heading, CTA
  • Feature list — 2/3-col grid or list of cards
  • Social — icon row (linkedin, twitter, etc.)

Content + media

  • Heading — H1 / H2 / H3 / H4
  • Text — rich-text body
  • Button — primary / secondary / ghost variants
  • Image — width %, alignment, optional link
  • HTML — raw escape hatch
  • Form embed — drop in any published Form by ID
  • Divider · Spacer — layout primitives

If something feels off

  • Hard-refresh first: ⌘ ⇧ R or Ctrl ⇧ R. The latest deploy may not have loaded.
  • Pinned items missing — pinned/visited state is per-browser. Different device = empty.
  • Landing page not live — Status must be PUBLISHED, slug must be set, URL is /lp/<slug> not /landing-pages/.
  • Email goes to spam — DKIM / SPF on sender domain. Settings → Email sender domain.
